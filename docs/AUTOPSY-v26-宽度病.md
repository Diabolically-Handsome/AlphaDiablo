# v26-leg6 宽度病尸检报告(2026-07-11,发射失败归因)

材料:满 32 逐种子档案 `v26-G3-leg6.json`(108.2,配对 +16.2,赢 11/32)对
`v24-G3-leg7.json`(92.0,历版同锚存档)。解剖刀:逐种子配对 + 桶级行为学
(farm 窗数 / 窗均 τ / 击杀 / mode 组成 / 死亡)。

## 一、三块病理组织

**赢桶(11 种子,配对差均值 +83.2,中位 +66.3)——驻窗深耕**:
v26 在富图上改打"少窗长驻":farm 窗数 55.6 对存档 78.2,窗均 τ 74.3 对
48.4,击杀 63.3 对 34.1。极端案例 7031(16 窗@τ̄174 对 73 窗@τ̄40,
367.4 对 122.4,+245)、7002(17 窗@τ̄152 对 85 窗@τ̄34,+229.5)。
**战略是真的,不是评测噪声。**

**病灶一(6 种子,共 −346.4,占全部亏损额的 87%)——读图误判**:
- 7008(−150.8):v26 战死(存档活得好好的),击杀 6 对 65——富图开局即溃,
  三例死亡中唯一的"新增死"。
- 7011(−61.0):全局 107 个 farm 窗仅 1 杀(存档 26 杀)——完全不接敌,游魂图。
- 7017(−71.9)/7010(−28.9)/7009(−17.2)/7013(−16.6):**该驻不驻**——
  在这些图上 v26 反而窗多 τ 短(7017:86 窗@τ33 对存档 45 窗@τ56),
  把富图误读成贫图。
- 方向性:图判读双向出错——富图读成贫图(不驻),个别图完全不接敌。

**病灶二(15 种子,中位 −4.2,带宽 [−9.6, −0.4])——普通图探图摩擦**:
输桶整体窗多(86.7 对 77.5)、杀少(34.7 对 41.6)、τ 略短。策略在普通图上
付出小额"探富"成本。单看是噪声级,但发射线数的是赢数——15 张摩擦图全记败。

## 二、算术含义(发射线视角)

- 均差 +16.2 早已达标;卡死在赢 11/32 < 18。
- 修好病灶一(6 图打回平手)只把均值推到 ~119,赢数纹丝不动仍 11——
  **发射的钥匙在病灶二**:摩擦图要从 −4 翻成 +ε,至少翻 7 张。
- 即:宽度病 = 大胜策略在普通图上的小额执行税,税率 −4 中位。翻面在能力
  射程内,首选假说是**欠训**(leg 4→5→6 = 98.6→112.2→114.5 未见平台)。

## 三、深度口径(总设计师"稳定打穿 4 层"愿望的现状测量)

满 32 深度分布:v26-leg6 = {0:2, 1:26, 2:4};存档、v27-leg7、v24-golden
全部相同(中位 1,l3=0)。**全文明住在 0-2 层,从未有系统到过第 3 层。**
机制:现行回报下,1 层驻窗刷怪的每 τ 收益远高于下潜(+8/层),冻结的 v22-H
经理据此几乎不选 DIVE(0-1/局)。**结论:4 层不是"练得不够"的问题,是
"目标不同"的问题**——宽度训练不会也不应改变深度;4 层 demo 需要下潜导向的
目标变体 = 环境侧改动,按铁律须总设计师亲批立项。
